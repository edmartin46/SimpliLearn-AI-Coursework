################################################################## Example Question 1 ##############################################################################

Question: What column types does monday.com support?
Answer: monday.com supports many built‑in column types (plus custom/app columns). Common column types include:

- Text (Short text)
- Long text / Updates
- Numbers (including Money)
- Formula
- Status
- Dropdown / Labels
- Tags
- Date (and Date & Time)
- Timeline
- Duration / Time tracking
- People (assignees)
- Auto‑number
- Checkbox
- Files (attachments)
- Link
- Email
- Phone
- Location
- Progress / Percent
- Rating
- Vote
- Color
- Dependency (predecessor/successor)
- Connect Boards (board relation) and Mirror (mirrored columns)
- Creation Log / Created By / Last Updated / Update Log (system columns)
- Subitems (special hierarchical rows)

There are also app‑provided or marketplace column types and additional "other types" used by the API (column capability objects, calculated capability objects, etc.). For the full, up‑to‑date list and API support/implementation details, see the monday.com columns documentation: https://developer.monday.com/api-reference/reference/columns-other-types

################################################################## Example Question 2 ##############################################################################


Question: How do I get a list of columns in a board using the monday.com API?
Answer: Use the GraphQL API and request the board's columns field. Columns must be requested nested under boards (you cannot query columns at the root). The request requires the boards:read scope.

Minimal GraphQL query
query {
  boards(ids: [123456789]) {
    columns {
      id
      title
      type
      settings_str
    }
  }
}

Example curl
curl -X POST https://api.monday.com/v2 \
  -H "Authorization: YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"query":"query { boards(ids: [123456789]) { columns { id title type settings_str } } }"}'

What you get
- The response contains an array of column metadata (id, title, type, etc.).
- If you also need column values for items, request items -> column_values in the same query.

Notes
- Required scope: boards:read.
- You can include other column fields (and column-specific fields) where supported.
- There is an optional columns(...) argument to filter by capabilities if you want only certain types of columns.
